# 추가 설정 가이드

현재 코드 수정만으로도 **포털 데이터 변경 시 캘린더가 기존 전체삭제 방식보다 안전하게 동기화**되도록 반영했습니다.

아래는 선택 사항입니다(필수 아님).

## 1) 동기화 주기 조정 (권장)
이미 GitHub Actions 자동 동기화를 사용 중이라면, 주기만 조정하면 됩니다.

- 권장: `SYNC_INTERVAL_HOURS=3` 또는 `SYNC_INTERVAL_HOURS=6`
- 더 자주 반영하고 싶으면 GitHub Actions cron 주기도 함께 짧게 조정

## 2) 실행 방식 정리
GitHub Actions 자동 동기화 방식에서는 **앱을 개인 PC에서 항상 켜둘 필요가 없습니다.**

- GitHub Actions가 정해진 스케줄마다 실행되어 자동 반영
- 수동 반영이 필요하면 `Run workflow`로 즉시 1회 실행 가능

## 3) 계정 권한
이미 확인하신 것처럼 서비스 계정에 캘린더 수정 권한이 있으면 정상 동기화됩니다.
